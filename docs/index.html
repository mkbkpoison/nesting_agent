🤖 套料软件智能助手

基于 Spring Boot 3 + Spring AI + RAG + Tool Calling + MCP 协议构建的企业级多智能体系统

📋 项目概述

套料软件智能助手是一个企业级AI系统，能够回答客户关于套料软件的问题，自动诊断后台错误，提供使用指导等功能。系统采用多智能体协作架构，支持意图识别、任务分发、工具调用和知识检索。

技术栈

Java 17 Spring Boot 3.3 Spring AI 1.0 PostgreSQL PGVector Redis Maven 百度文心/阿里通义

🔄 用户数据处理流程

1

用户发送请求

用户通过 REST API 发送消息，包含用户消息、会话ID（可选）、用户ID（可选）

⬇️
2

ChatController 接收请求

API层接收HTTP请求，调用 NestingAssistantService 进行处理

⬇️
3

会话管理

生成或恢复会话ID，将用户消息存入 Redis 对话记忆

⬇️
4

智能体编排 (AgentOrchestrator)

构建智能体上下文（AgentContext），恢复之前的对话状态，调用 RouterAgent

⬇️
5

意图识别 (RouterAgent)

分析用户消息，识别意图类型：技术问题/系统诊断/知识查询/操作执行/普通对话

⬇️
6

任务分发

根据意图类型，将请求分发到对应的专业智能体处理

⬇️
7

专业智能体处理

专业智能体调用 ChatClient，可能触发以下操作：
• Tool Calling: 调用诊断/检索/操作工具
• RAG检索: 从向量数据库检索相关知识
• LLM推理: 使用文心/通义模型生成回答

⬇️
8

响应生成

智能体生成结构化响应（AgentResponse），包含回答内容、调用的工具、意图类型等

⬇️
9

保存对话

将助手响应存入 Redis 对话记忆，保存当前意图到会话状态

⬇️
10

返回响应

返回 ChatResponse 给用户，包含回答消息、会话ID、处理的智能体角色等

🤝 多智能体角色

智能体 角色 职责 处理意图
Router Agent 路由智能体 意图识别、任务分发、多轮对话管理 所有请求入口
Technical Support 技术支持智能体 套料参数调优、配置检查、错误代码分析 TECHNICAL_QUESTION
System Diagnosis 系统诊断智能体 日志检查、数据库检测、资源监控、许可证检查 SYSTEM_DIAGNOSIS
Knowledge Retrieval 知识检索智能体 RAG知识库检索、相似问题匹配、解决方案推荐 KNOWLEDGE_QUERY
Operation 操作执行智能体 执行系统命令、生成诊断报告、配置备份 OPERATION

📦 模块结构

nesting-assistant-common

  • ApiResponse 统一响应
  • BusinessException 异常处理
  • GlobalExceptionHandler
  • IdGenerator, JsonUtils

nesting-assistant-domain

  • AgentRole, IntentType 枚举
  • Conversation 会话实体
  • ChatMessage 消息实体
  • KnowledgeDocument 知识文档

nesting-assistant-memory

  • ConversationMemoryService
  • ConversationStateService
  • Redis会话管理

nesting-assistant-rag

  • NestingDocumentService
  • NestingRagService
  • InitialKnowledgeLoader
  • PGVector向量存储

nesting-assistant-tools

  • 14个工具函数实现
  • 系统诊断工具(5个)
  • 文件操作工具(3个)
  • 知识/套料工具(6个)

nesting-assistant-agent

  • 5个智能体实现
  • AgentOrchestrator 编排器
  • AgentContext/Response
  • 意图识别与分发

🚀 快速开始

# 克隆项目
git clone <repository-url>
cd nesting-assistant

# 配置环境变量
export WENXIN_API_KEY=your-api-key

# 运行项目
./mvnw spring-boot:run -pl nesting-assistant-starter

# 访问接口
API文档: http://localhost:8080/swagger-ui.html
健康检查: http://localhost:8080/api/v1/diagnosis/health
